В данной директории описывается обработка http-сообщений в соответствии с протоколом http (поддерживается часть функционала).
Сервер парсит приходящие запросы по следующему формату (в соотв. с HTTP/1.1):

$method $uri HTTP/1.1
header_1: value_1
...
header_n: value_n

$body

То есть запрос разбивается на строки (разделенные "\r\n").
Когда считана пустая строка перед $body, первая строка и header-строки проверяются на соответствие шаблону:
1) Первая строка начинается с "$method ", где $method - поддерживаемый сервером метод, а заканчивается на HTTP/1.1
2) Header-строки содержат ':'

После этого запрос обрабатывается согласно файлу "$method.txt".
Такой файл должен определять: 
1) Какие заголовки запроса будут использованы для обработки запроса. 
Для этих заголовков вводятся переменные с таким же именем (например: Host -> $host), которые имеют значение NULL если заголовок не содержится в запросе, иначе равны значению после ': '.
Остальные заголовки игнорируются (даже если таких заголовках нет в стандарте HTTP) (add_info.txt, п.2)
2) Алгоритм валидации заголовков вместе с $uri.
В процессе алгоритма могут определятся новые переменные и возвращаться ошибки (в этом случае запрос дальше не читается).
3) Алгоритм дочитывания тела сообщения $body (возможно, на основе данных из п.2).
4) Алгоритм формирования запроса, после того как запрос дочитан.
